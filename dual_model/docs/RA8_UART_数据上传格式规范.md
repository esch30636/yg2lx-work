# RA8 → RZ/G2L UART 数据上传格式规范

> **接收方:** RZ/G2L (MYD-YG2LX, Linux), 推荐 `/dev/ttySC3` (UART3/SCIF3)  
> **发送方:** RA8 微控制器 (裸机, e2studio FSP)  
> **物理接口:** UART, 2 线 (TX + GND)  
> **波特率:** 115200, 8N1 (8 data bits, no parity, 1 stop bit)  
> **更新频率:** 每 100ms 发送一帧（10 Hz）  
> **文档版本:** v2.0, 2026-07-10  
>
> **⚠ 设备名以板端实际查询为准。** 上板先跑：
> ```
> ls /dev/ttySC* && dmesg | grep -Ei "tty|uart|serial|scif"
> ```
> 根据输出确认实际设备名。`/dev/ttySC3` 是 UART3 (SCIF3) 的预期名称。  

---

## 1. 硬件接线

```
RA8                          RZ/G2L J20 40-pin 排针
════                         ═══════════════════════
TX  (UART TXD)  ──────────── UART3_RX  (P0_1, SCIF3 RXD)
GND             ──────────── GND

只需要两根线。
UART3 = SCIF3 = /dev/ttySC3 (内核已启用)。
引脚: P0_0 = TXD, P0_1 = RXD, Function 5。
在 J20 排针上的具体物理位置需查 MYC-YG2LX-Pin List 文档（米尔开发者中心下载）
或通过 GPIO sysfs 翻转电平 + 万用表定位。
```

## 2. 数据格式

### 一行一帧，逗号分隔，换行结尾。

```
电压,电流,温度,压力\n
```

**示例（实际数据）：**

```
3.652,1.234,25.6,1.201
3.648,1.241,25.7,1.198
3.655,1.228,25.6,1.203
```

### 字段定义

| 顺序 | 名称 | 类型 | 单位 | 范围 | 分辨率 | 说明 |
|:----:|------|------|------|------|:------:|------|
| 1 | 电池电压 | float | V（伏特） | 0.00 – 5.00 | 0.001V | 电芯正负极间电压 |
| 2 | 电流 | float | A（安培） | -100.0 – 100.0 | 0.001A | **正数=充电，负数=放电** |
| 3 | 温度 | float | °C（摄氏度） | -20.0 – 80.0 | 0.1°C | 电芯表面温度 |
| 4 | 压力 | float | V（伏特） | 0.00 – 5.00 | 0.001V | FSR/压力传感器原始电压 |

### 格式约束（严格遵守）

1. **四个字段缺一不可。** 每个字段必须存在，不允许空值。
2. **固定顺序。** 电压,电流,温度,压力。不可调换。
3. **逗号分隔。** ASCII 逗号 `,` (0x2C)，无空格。
4. **换行结尾。** 每帧以 `\n` (LF, 0x0A) 结尾。不要用 `\r\n`。
5. **浮点格式。** 电压/电流/压力保留 3 位小数，温度保留 1 位小数。用 `%.3f` / `%.1f`。
6. **纯 ASCII。** 不发送任何二进制数据、汉字、控制字符（除末尾 `\n`）。
7. **无帧头帧尾。** 一行就是一帧，不需要额外的起始/结束标记。
8. **每行不超过 64 字节。** 实际约 25–30 字节，远低于此上限。

## 3. RA8 端实现

### C 代码（在 1ms 主循环中）

```c
/* ═══════════════════════════════════════════════════════════════════
 * RA8 UART 数据发送
 * 在 hal_entry.c 的 1ms 主循环中添加以下代码。
 *
 * 前提:
 *   - UART 已初始化 (假设为 g_uart0, SCI0)
 *   - 全局变量 g_voltage_v, g_current_a, g_temperature_c, g_pressure_v
 *     由 ADC 采样 + 卡尔曼滤波后更新（你现有代码已有）
 * ═══════════════════════════════════════════════════════════════════ */

/* 全局变量 — 你代码里应该已经有这些 */
extern float g_voltage_v;       /* 电池电压, V */
extern float g_current_a;       /* 充电电流, A (正=充电, 负=放电) */
extern float g_temperature_c;   /* 电池温度, °C */
extern float g_pressure_v;      /* FSR 压力传感器, V */

/* 发送计数器 — 1ms 主循环中每 100 次发一帧 = 10Hz */
static uint32_t send_tick = 0;

/* 放在你的 1ms 主循环中 */
send_tick++;
if (send_tick >= 100) {
    send_tick = 0;

    char buf[64];
    int len = snprintf(buf, sizeof(buf),
                       "%.3f,%.3f,%.1f,%.3f\n",
                       (double)g_voltage_v,
                       (double)g_current_a,
                       (double)g_temperature_c,
                       (double)g_pressure_v);

    /* 发送 — 使用你现有的 UART 发送函数 */
    R_SCI_UART_Write(&g_uart0, (uint8_t *)buf, (uint32_t)len);
}
```

### 关键要求

- **不要在 UART 发送中加锁或关中断。** `snprintf` + `R_SCI_UART_Write` 总共约 20–50 微秒，对 1ms 主循环无影响。
- **不要等 UART 发送完成。** FSP 的 `Write` 函数是阻塞的，但对 30 字节 @ 115200 来说约 2.6ms，不会影响你的 ADC/PID 时序（你的循环是 1ms，但一帧发送完才继续，实测约 3ms 完成一帧发送，相当于偶尔多花 2ms）。
- **不要在中断里发。** 在 1ms 主循环的普通代码路径里发。

### 电流符号约定（重要）

```
充电: g_current_a > 0 → 输出正数, 如 "1.234"
放电: g_current_a < 0 → 输出负数, 如 "-1.234"
静置: g_current_a ≈ 0 → 输出 "0.000"
```

如果你的代码里电流变量是绝对值（总是正数），请在 `snprintf` 前根据充放电状态加上符号：

```c
float signed_current = (g_charge_state == CHARGING) ? g_current_a : -g_current_a;
```

## 4. RZ/G2L 端接收

### 第 0 步：确认设备名（每次上板必须先做）

```bash
# 列出所有串口设备
ls /dev/ttySC* /dev/ttyS* /dev/ttyAMA* 2>/dev/null

# 查看内核串口日志，确认每个设备对应哪个 SCIF
dmesg | grep -Ei "tty|uart|serial|scif"
```

根据输出确认实际设备名。以下为 MYD-YG2LX 预期映射：

| 设备名 | GPIO | 状态 | 说明 |
|--------|------|:---:|------|
| `/dev/ttySC0` | P38_0, P38_1 | 占用 | 调试控制台 (接电脑) |
| `/dev/ttySC1` | P40_0, P40_1 | 空闲 | RS485 (无子板时空闲) |
| `/dev/ttySC2` | P48_0, P48_1 | 空闲 | RS232 (无子板时空闲) |
| **`/dev/ttySC3`** | **P0_0, P0_1** | **空闲** | **纯 UART，推荐使用** |

### 一行命令验证（以 ttySC3 为例）

```bash
# 配置串口 — 把 /dev/ttySC3 换成你实际查到的设备名
stty -F /dev/ttySC3 115200 cs8 -cstopb -parenb -echo raw

# 直接看数据
cat /dev/ttySC3
```

## 5. 常见错误对照

| 错误示例 | 问题 | 正确写法 |
|----------|------|----------|
| `3.65 1.23 25.6 1.20` | 空格分隔 | `3.650,1.230,25.6,1.200` |
| `V:3.65,A:1.23,...` | 加了字段名 | 不要加标签 |
| `3.650,1.230,25.6,1.200\r\n` | Windows 换行 | 只用 `\n` |
| `3.65,1.23,25.6` | 缺少压力字段 | 必须 4 个字段 |
| `3650,1230,25.6,1.20` | 电压电流单位错误 | 电压单位是 V 不是 mV |
| 发送间隔 2 秒以上 | 频率太低 | 至少 10Hz |
| 发送二进制 float | 不可读 | 发送 ASCII 文本 |
| `3.650,,25.6,1.200` | 空字段 | 每个字段必须有值 |

## 7. 故障排查

| 症状 | 检查项 |
|------|--------|
| RZ 端 `cat` 无输出 | 1. TX 和 RX 是否交叉连接<br>2. 波特率是否一致 (两端都是 115200)<br>3. GND 是否共地 |
| 出现乱码 | 1. 波特率不匹配<br>2. 数据位/停止位/校验位不一致 (应为 8N1)<br>3. 接地不良 |
| 数据断断续续 | 1. 检查 RA8 是否真的每 100ms 发一次<br>2. 检查 UART 发送是否被中断阻塞 |
| 数值对不上 | 1. 检查单位 (V 不是 mV)<br>2. 检查电流符号 (充电正/放电负)<br>3. 检查 ADC 分压比校准 |
| 偶尔丢一行 | 1. 可能是 `snprintf` 缓冲区溢出 (buf[64] 足够)<br>2. RZ 端 `cat` 进程被其他串口读取竞争 |

---

> **一句话总结:**  
> RA8 每 100ms 通过 UART (115200, 8N1) 发送一行 `电压,电流,温度,压力\n`，逗号分隔，纯 ASCII。只此一条指令，不需任何协议栈。
