## 2. 传什么数据

### 就 4 个数

RA8 通过传感器采集到 4 个物理量，需要传给 RZ/G2L：

| 序号 | 叫什么 | 英文名 | 单位 | 从哪来 | 举例 |
|:----:|--------|--------|------|--------|------|
| 1 | 电池电压 | Voltage | V（伏特） | 分压电阻 + ADC | 3.652 |
| 2 | 电流 | Current | A（安培） | 采样电阻 + ADC | 1.234 |
| 3 | 温度 | Temperature | °C（摄氏度） | 热敏电阻/NTC + ADC | 25.6 |
| 4 | 压力 | Pressure | V（伏特） | FSR 压力传感器 + ADC | 1.201 |

**一次发送就是这一行：**

```
3.652,1.234,25.6,1.201
```

- 4 个数字用英文逗号 `,` 隔开
- 末尾加一个回车换行 `\n`
- 纯文本，人能直接读

### 多久发一次

**每 100 毫秒发一次。** 即每秒发 10 次（10 Hz）。

```
时间 →
0ms        100ms       200ms       300ms
│           │           │           │
├─ 发一行 ──├─ 发一行 ──├─ 发一行 ──├─ ...
│           │           │           │
3.652,...  3.648,...   3.655,...   3.644,...
```

### 为什么是这个频率

RZ/G2L 上的 HMI 程序每 100ms 读一次数据。发太快浪费，发太慢画面卡。

### 数据格式的硬性规定

这 7 条是给写 RA8 代码的人看的，一条都不能违反：

1. **4 个字段，不能少。** 缺字段 RZ 端解析会出错。
2. **固定顺序。** 电压 → 电流 → 温度 → 压力。不能调换。
3. **用逗号分隔。** `3.652,1.234,25.6,1.201`。不能用空格、分号、Tab。
4. **换行用 `\n`。** 不能用 `\r\n`（Windows 格式）。
5. **电压/电流/压力保留 3 位小数。** `%.3f`。温度保留 1 位小数。`%.1f`。
6. **纯英文和数字。** 不要发中文、二进制、控制字符。
7. **一行不超过 64 个字符。** 实际约 25 个，大量余量。

## 3. 怎么接线

### 需要的东西

- 2 根杜邦线（母对母），淘宝 2 块钱
- RA8 和 RZ/G2L 共地（GND 连在一起）

### 接线图

```
RA8 板                          MYD-YG2LX 开发板
══════                          ════════════════

找一个 GND 引脚 ─────────────── 找一个 GND 引脚
                                   (黑色线)

找一个 UART TX 引脚 ──────────── 找一个 UART RX 引脚
  (发送)                            (接收, 串口5)
                                    (白色或绿色线)
```

### 线的颜色（建议）

| 颜色 | 用途 |
|------|------|
| 黑色 | GND（地线） |
| 白色或绿色 | TX → RX（数据线） |

**只需要 2 根线。** 不需要时钟线、片选线、电源线。

### 在 RA8 上找 UART TX 引脚

查阅你的 RA8 开发板原理图或引脚定义表，找到 SCI（串行通信接口）的 TXD 引脚。常见命名：

- `TXD0` / `TXD1` / `TXD5`
- `SCI0_TX` / `SCI1_TX`
- `Pxxx`（具体引脚号因型号而异）

### 在 MYD-YG2LX 上找 UART RX 引脚

MYD-YG2LX 有 4 路 SCIF 串口（内核源码已验证）。空闲可用的是：

| 串口 | Linux 设备名 | GPIO (TXD, RXD) | 状态 |
|------|-------------|------------------|:---:|
| SCIF0 | `/dev/ttySC0` | P38_0, P38_1 | ❌ 调试控制台 (被电脑占用) |
| SCIF1 | `/dev/ttySC1` | P40_0, P40_1 | ✅ 空闲 (无 MY-WiredCom 子板时) |
| SCIF2 | `/dev/ttySC2` | P48_0, P48_1 | ✅ 空闲 (无 MY-WiredCom 子板时) |
| **SCIF3** | **`/dev/ttySC3`** | **P0_0, P0_1** | **✅ 空闲，推荐使用** |

> **推荐 `/dev/ttySC3` (UART3/SCIF3)。** 它是唯一纯 UART 接口（不像 SCIF1/2 被定义为 RS485/RS232），出厂内核已启用 (`status = "okay"`)，引脚 P0_0(TXD)/P0_1(RXD) 在 J20 排针上。

**⚠ 设备名以板端实际查询为准。** 上板先跑：
```bash
ls /dev/ttySC* && dmesg | grep -Ei "tty|uart|serial|scif"
```

具体在 J20 排针上的物理位置需查 `MYC-YG2LX-Pin List` 文档（米尔开发者中心下载），或通过 GPIO sysfs 翻转 + 万用表定位。

## 4. 怎么传：UART 通信原理

### 什么是 UART

UART 是最简单的通信方式之一。两个设备约定好：

1. **通信速度**（波特率）：双方用相同的速度，这里是 115200
2. **数据格式**：8 个数据位，无校验位，1 个停止位（简称 8N1）

通信过程：

```
RA8 想发 "3.652,1.234,25.6,1.201\n" 这 26 个字符

RA8 TX 引脚:   ────┐     ┌─┐ ┌─┐     ┌───
                   │     │ │ │ │     │
                   └─────┘ └─┘ └─────┘
                  idle   '3'  '.'  '6'  ...一个一个字符发出去
                        (0x33)(0x2E)(0x36)
                  
RZ/G2L RX 引脚: 收到完全相同的波形
                ↓
          Linux 内核把字符拼起来
                ↓
          程序读到 "3.652,1.234,25.6,1.201\n"
```

### 波特率 115200 意味着什么

```
115200 波特 = 每秒传 115200 个二进制位

一个字符 = 1 起始位 + 8 数据位 + 1 停止位 = 10 位
每秒字符数 = 115200 / 10 = 11520 个字符/秒

我们每帧约 26 个字符 × 每秒 10 帧 = 260 个字符/秒
带宽利用率 = 260 / 11520 = 2.3%

结论：带宽大把富裕，完全不用担心不够用。
```

### UART vs SPI 对比（为什么要用 UART）

| | UART | SPI |
|---|---|---|
| 线数 | **2 根** | 4~6 根 |
| RA8 端代码量 | **6 行** | 100+ 行 |
| 会不会打断 ADC/PID | **不会** | 可能 |
| 要不要修改 Linux 设备树 | **不要** | 可能需要 |
| 调试工具 | **串口助手直接看** | 需要逻辑分析仪 |
| 线长要求 | 随便（杜邦线几十厘米都行）| 短（≤10cm）+ 终端电阻 |
| 带宽 | 较低（但对本项目 **完全够用**） | 高（本项目 **用不上**） |

## 5. RA8 端要写什么代码

### 前提确认

在写发送代码之前，确认 RA8 上已经：

1. ✅ ADC 能正确读到电压、电流、温度、压力
2. ✅ 卡尔曼滤波正常工作（你已有的代码）
3. ✅ PID 控制正常工作（你已有的代码）
4. ✅ 有一个 `1ms` 的主循环或定时器

### 第 1 步：初始化 UART

在 `hal_entry.c` 的初始化部分（`setup()` 或 `init()` 函数中），添加 UART 初始化：

```c
/* ═══════════════════════════════════════════════════════
 * UART 初始化 (在 hal_entry.c 的初始化部分添加)
 * ═══════════════════════════════════════════════════════ */

/* FSP 会自动生成 UART 初始化代码。
   在 e2studio 的 FSP Configuration 中：
   
   1. 打开 Stacks 标签页
   2. 添加一个 "SCI UART" 模块
   3. 配置:
      - Channel: 选一个空闲的 SCI 通道
      - Baud Rate: 115200
      - Data Bits: 8
      - Parity: None
      - Stop Bits: 1
      - Flow Control: None
   4. 记下生成的 UART 句柄名, 通常是 g_uart0
   
   如果你已有 UART 初始化代码, 确认以上参数正确即可。 */
```

### 第 2 步：在 1ms 主循环中加发送代码

```c
/* ═══════════════════════════════════════════════════════
 * 在 hal_entry.c 的 1ms 主循环 while(1) 中添加以下代码
 * ═══════════════════════════════════════════════════════ */

/* ══ 假设你已有这些全局变量 (ADC + 卡尔曼滤波后的值) ══ */
extern float g_battery_voltage;    /* 电池电压, 单位 V */
extern float g_battery_current;    /* 电流, 单位 A, 正=充电, 负=放电 */
extern float g_battery_temp;       /* 温度, 单位 °C */
extern float g_fsr_pressure;       /* FSR 压力, 单位 V */

/* ══ 新加: 发送计数器 ══ */
static uint32_t g_uart_send_tick = 0;

/* ══ 你已有的 1ms 主循环 ══ */
void main_loop_1ms(void)
{
    /* ... 你已有的 ADC 采样代码 ... */
    /* ... 你已有的卡尔曼滤波代码 ... */
    /* ... 你已有的 PID 控制代码 ... */

    /* ──────────────────────────────────────────────────
     * 以下是新加的 UART 发送代码 (共 7 行)
     * ────────────────────────────────────────────────── */
    g_uart_send_tick++;
    if (g_uart_send_tick >= 100)          /* 每 100 次 = 100ms */
    {
        g_uart_send_tick = 0;

        char buf[64];
        int len = snprintf(buf, sizeof(buf),
                           "%.3f,%.3f,%.1f,%.3f\n",
                           (double)g_battery_voltage,
                           (double)g_battery_current,
                           (double)g_battery_temp,
                           (double)g_fsr_pressure);

        R_SCI_UART_Write(&g_uart0,        /* 你的 UART 句柄 */
                         (uint8_t *)buf,
                         (uint32_t)len);
    }
    /* ─────────── UART 发送代码结束 ─────────── */
}
```

### 完整代码就是这 7 行

不需要：
- ❌ 中断服务函数
- ❌ DMA 配置
- ❌ 帧协议
- ❌ CRC 校验
- ❌ 二进制打包
- ❌ 缓冲区管理

### 注意事项（给 RA8 程序员）

1. **电流的正负号很重要：**
   ```
   充电时：g_battery_current > 0  → 输出 "1.234"
   放电时：g_battery_current < 0  → 输出 "-1.234"
   静置时：g_battery_current ≈ 0  → 输出 "0.000"
   ```
   如果你的代码里电流是绝对值（永远正数），需要根据充放电状态加上符号：
   ```c
   float signed_current = (g_is_charging) ? g_battery_current : -g_battery_current;
   ```

2. **不要在中段服务函数里发 UART。** 在普通的主循环代码路径里发。

3. **`snprintf` 的参数类型是 `double`。** 如果你的变量是 `float`，用 `(double)` 强转一下。

4. **UART 发送是阻塞的。** 发 26 个字符约需 2.6ms，这段时间 CPU 在等。这对 1ms 循环影响很小——本来 100ms 才发一次，多发 2.6ms 无影响。

## 6. RZ/G2L 端怎么收数据

### 第 1 步：确认串口设备存在（必须先做）

在 RZ/G2L 的 Linux 终端执行：

```bash
# 列出所有串口设备
ls /dev/ttySC* /dev/ttyS* /dev/ttyAMA* 2>/dev/null

# 查看内核日志，确认各设备对应的 SCIF 通道
dmesg | grep -Ei "tty|uart|serial|scif"
```

根据输出确认你的目标串口设备名。以下为预期映射（内核源码已验证）：

| 设备名 | TX GPIO | RX GPIO | 状态 |
|--------|---------|---------|:---:|
| `/dev/ttySC0` | P38_0, Func1 | P38_1, Func1 | 调试控制台，不可用 |
| `/dev/ttySC1` | P40_0, Func1 | P40_1, Func1 | 空闲 |
| `/dev/ttySC2` | P48_0, Func1 | P48_1, Func1 | 空闲 |
| `/dev/ttySC3` | P0_0, Func5 | P0_1, Func5 | **空闲，推荐** |

记下你选用的设备名，后续命令中的 `/dev/ttySC3` 都要替换为实际设备名。

### 第 2 步：直接用 cat 命令验证数据

这是最快验证 RA8 是否在发数据的方法：

```bash
# 把 /dev/ttySC3 换成你第 1 步查到的实际设备名
# 配置串口参数
stty -F /dev/ttySC3 115200 cs8 -cstopb -parenb -echo raw

# 读数据
cat /dev/ttySC3
```

如果 RA8 在正常发送，你会看到：

```
3.652,1.234,25.6,1.201
3.648,1.241,25.7,1.198
3.655,1.228,25.6,1.203
3.644,1.239,25.8,1.205
...
```

**一行一行不停地冒出来，就说明 RA8→RZ 通信成功了。**

### 第 3 步：理解数据到了 RZ/G2L 之后怎么走

```
                     RZ/G2L Linux 系统
                     ═══════════════════
                     
RA8 UART TX ────物理线──→ RX 引脚
                            │
                     Linux 内核 UART 驱动
                            │
                     /dev/ttySC3 (像文件一样读)
                            │
                     ┌──────┴──────┐
                     │              │
                  cat 命令      battery_hmi 程序
               (临时测试用)    (正式使用)
                     │              │
                     ▼              ▼
               屏幕打印出来     UartDataProvider
                              (读串口 → 转成
                               BatterySample →
                               输入 AI 模型)
                                   │
                                   ▼
                              PINN/CNN 推理
                                   │
                                   ▼
                              HMI 屏幕显示
```

`/dev/ttySC3` 在 Linux 里就像一个特殊的"文件"。读它就是在读串口收到的数据。

## 7. 数据到达模型的全流程

### 完整链路（一步一步）

```
步骤1: 电池传感器
  │    电压(分压电阻) → ADC
  │    电流(采样电阻) → ADC  
  │    温度(NTC热敏电阻) → ADC
  │    压力(FSR传感器) → ADC
  │
  ▼
步骤2: RA8 芯片 (裸机, e2studio)
  │    ┌─ ADC 读取 12-bit 原始值
  │    ├─ 物理量换算 (V, A, °C)
  │    ├─ 卡尔曼滤波 (降噪)
  │    ├─ PID 控制 (充放电保护)
  │    └─ 每 100ms:
  │        snprintf("3.652,1.234,25.6,1.201\n")
  │        UART TX → 发送
  │
  ▼  两根杜邦线
  │
步骤3: RZ/G2L 开发板 (Linux, Yocto)
  │    UART RX 引脚收到数据
  │    Linux 串口驱动 → /dev/ttySC3
  │
  ▼
步骤4: UartDataProvider (C++ 程序)
  │    ┌─ 每 100ms 读 /dev/ttySC3
  │    ├─ 解析 "3.652,1.234,25.6,1.201\n"
  │    │   → voltage=3.652, current=1.234, temp=25.6, pressure=1.201
  │    ├─ 累积历史数据 (电压、电流、容量随时间变化)
  │    ├─ 检测充放电循环
  │    ├─ 计算 dQ/dV 曲线 (128 点)
  │    └─ 填充 BatterySample 结构体
  │
  ▼
步骤5: AI 推理引擎 (纯 C)
  │    ┌─ PINN (3.5ms): features[132] → SOH
  │    └─ CNN (45ms): ic_curve[256] → 阶段 + RUL
  │
  ▼
步骤6: HMI 屏幕显示 (Qt5)
       ┌─ 示波器: 电压/电流曲线 (60秒滚动)
       ├─ SOH 值 + 置信区间
       ├─ 老化阶段 (健康/衰退/寿命终止)
       ├─ RUL 剩余寿命
       ├─ 循环次数
       └─ 报警 (过温/过压/膨胀/SOH过低)
```

### 关键理解

**RA8 只负责传 4 个原始数。** 它不需要知道：
- dQ/dV 怎么算
- 特征向量是什么
- AI 模型在跑什么

**RZ/G2L 负责所有的"智能"部分：**
- 从原始数据推导 IC 曲线
- 生成 132 维特征向量
- 跑 AI 模型推理
- 显示结果

## 8. 完整调试步骤

### 8.1 先验证 RA8 端

```c
/* 在 RA8 上先用固定值测试 UART 是否正常 */
char test[] = "3.650,1.200,25.0,1.000\n";
R_SCI_UART_Write(&g_uart0, (uint8_t *)test, strlen(test));
```

用示波器或 USB 转串口模块接到 RA8 的 TX 脚，看能不能在 PC 串口助手收到 `3.650,1.200,25.0,1.000`。

### 8.2 再验证物理连线

用万用表蜂鸣档：
- RA8 GND ↔ RZ/G2L GND：应该响（连通）
- RA8 TX ↔ RZ/G2L RX：应该响（连通）

### 8.3 验证 RZ/G2L 能收到

```bash
# RZ/G2L 终端
stty -F /dev/ttySC3 115200 cs8 -cstopb -parenb -echo raw
cat /dev/ttySC3
```

## 9. 常见问题

### 问题 1：`cat /dev/ttySC3` 没有任何输出

可能原因（按优先级排列）：

1. **RA8 没在发数据。** 检查 RA8 代码里 `g_uart_send_tick >= 100` 的逻辑是否正确。
2. **串口号不对。** 用 `ls /dev/tty*` 看有哪些串口，逐个试。
3. **TX 和 RX 接反了。** RA8 的 TX 必须接 RZ/G2L 的 RX。
4. **RA8 UART 初始化失败。** 检查 FSP 配置，确认 SCI UART 模块已启用。
5. **该串口被其他程序占用。** 用 `fuser /dev/ttySC3` 查看。

### 问题 2：收到乱码

1. **波特率不匹配。** RA8 和 RZ 两端必须都是 115200。
2. **数据格式不匹配。** 必须都是 8N1（8 数据位、无校验、1 停止位）。
3. **GND 没接或接触不良。**

### 问题 3：数据偶尔丢一行

1. **RA8 端的 1ms 循环被打断太久。** 检查是否有耗时超过 100ms 的操作（如 Flash 写入）。
2. **`snprintf` 缓冲区不够。** `buf[64]` 对 26 个字符绰绰有余——不会是这个问题。

### 问题 4：分压比/传感器校准值变了，数据不准

这和通信无关。检查 RA8 端的 ADC 换算系数和卡尔曼滤波参数。

## 附录 B：RA8 端完整代码模板

把下面这段代码发给 RA8 程序员，他一看就懂：

```c
/* ═══════════════════════════════════════════════════════════
 * 电池数据 UART 上报
 * 
 * 硬件: SCI0 (或其它 SCI 通道), TX 引脚, 115200 8N1
 * 频率: 每 100ms 发一帧
 * 格式: "电压,电流,温度,压力\n"   (逗号分隔, 换行结尾)
 * 
 * 在你的 1ms 主循环里调用 uart_send_battery_data()
 * ═══════════════════════════════════════════════════════════ */

#include <stdio.h>
#include <string.h>
#include "hal_data.h"    /* FSP 生成的 HAL 配置, 包含 g_uart0 */

/* 全局电池数据 — 由 ADC+卡尔曼更新 */
float g_bat_v = 0.0f;   /* 电压 (V) */
float g_bat_i = 0.0f;   /* 电流 (A), 正=充电, 负=放电 */
float g_bat_t = 25.0f;  /* 温度 (°C) */
float g_bat_p = 0.0f;   /* 压力 (V) */

static void uart_send_battery_data(void)
{
    char buf[64];
    int len = snprintf(buf, sizeof(buf),
                       "%.3f,%.3f,%.1f,%.3f\n",
                       (double)g_bat_v,
                       (double)g_bat_i,
                       (double)g_bat_t,
                       (double)g_bat_p);

    R_SCI_UART_Write(&g_uart0, (uint8_t *)buf, (uint32_t)len);
}

/* ══ 在你的 1ms 主循环中调用 ══ */
void main_loop_1ms(void)
{
    static uint32_t tick = 0;

    /* ... 你已有的 ADC / 卡尔曼 / PID ... */

    tick++;
    if (tick >= 100) {
        tick = 0;
        uart_send_battery_data();   /* ← 就这一行 */
    }
}
```
